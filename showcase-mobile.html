Mobile Showcase

React Native CLI

The main technology I used for developing mobile apps is React Native CLI. React Native CLI is the basic environment setup for developing application using React Native. The main advantages of using React Native over native platforms (Android Java / Kotlin & iOS Swift) are the quality of development speed and littler cost because of the use of a single code base.

Information System Mobile Apps.

I am experienced with developing a system information mobile application that includes basic functionalities such as: authentication, state / local data management (redux / context), connecting to API (REST API & GraphQL Client), styling & responsiveness, etc.

Third Party Integration

In developing mobile apps nowadays, there are many necessities that can be fufilled using third party library, module, or SDK. Here are some well-known third parties that I've experienced in integrating to my React Native project:

  • - Google Firebase (analytics, crashlytics & cloud messaging / notification)
  • - Google Maps (emit location & display route)
  • - Twilio (video call)
  • - SendBird (chat)
  • - Gqlgen (GraphQL typeSafe API)

Clean Code Implementation.

In the process of developing mobile application using React Native, the practice of clean code comes in many forms such as variable naming standard, applying linter rules, and most importantly by applying component or modulized coding structure and functions that prevents cluttered codes based on Clean Code Principles and React Native Optimization Guide.
